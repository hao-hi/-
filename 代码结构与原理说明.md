# 代码结构与原理说明

## 1. 总体分层

这个工程可以按 4 层理解：

1. 数学与物理基础层
2. 传感器、滤波与辨识层
3. 控制与优化层
4. 实验组织与输出层

对应文件如下：

### 1.1 数学与物理基础层

- `core_utils.py`
- `dynamics.py`

### 1.2 传感器、滤波与辨识层

- `startracker.py`
- `mekf.py`
- `estimators.py`

### 1.3 控制与优化层

- `controller.py`
- `adrc_controller.py`
- `optimizers.py`

### 1.4 实验组织与输出层

- `main.py`
- `visualization.py`
- `benchmark.py`

## 2. 核心模块职责

### 2.1 `main.py`

这是项目的实验调度中心，负责：

- 运行档位管理：`_resolve_run_profile(...)`
- 单次闭环仿真：`simulate_attitude_control(...)`
- PD 自动调参：`compare_pd_gain_optimizers(...)`
- 输出目录与报告组织
- 批量保存图像、表格和总结

当前还包含两个值得注意的辅助能力：

- `result_mode='metrics'`
  - 用于 PD 调参时的轻量评估
  - 只保留目标函数所需指标
- 随机量批量预生成
  - 降低主循环内的随机采样开销

### 2.2 `dynamics.py`

实现卫星刚体姿态动力学。

状态：

- 四元数 `q`
- 角速度 `w`

输入：

- 控制力矩 `u`
- 扰动力矩 `dist`

特点：

- 使用 RK4 积分
- 支持在线更新惯量 `update_inertia(new_J)`
- 对对角惯量走快速分支，减少热路径中的矩阵运算

### 2.3 `controller.py`

实现四元数误差下的 PD 控制律：

`u = -Kp * q_e[1:] - Kd * w`

只负责 PD 本体，不再重复定义四元数误差等公共数学函数。

### 2.4 `adrc_controller.py`

实现三轴并行 LADRC。

包含三部分：

- TD：平滑参考姿态
- ESO：估计误差状态与总扰动
- Feedback：输出控制力矩

同时支持：

- `update_b0(new_b0)` 在线更新控制增益
- `get_disturbance_estimate_torque()` 输出等效力矩扰动估计

### 2.5 `mekf.py`

实现乘法扩展卡尔曼滤波。

主要类：

- `MEKFBiasOnly`
  - 误差状态 `[δθ, δb]`
  - 负责姿态与陀螺偏置估计
- `MEKF_Augmented`
  - 误差状态 `[δθ, δb, ΔJ]`
  - 额外在线估计对角惯量偏差

### 2.6 `estimators.py`

负责对角惯量辨识相关工具。

主要内容：

- `ensure_diag_inertia(...)`
- `build_inertia_regression_matrix(...)`
- `diagonal_inertia_wdot_jacobian(...)`
- `InertiaRLS`

`InertiaRLS` 当前加入了较多稳定化措施，例如：

- 激励不足冻结
- 创新裁剪
- 单步参数更新限幅
- 协方差上下界投影

### 2.7 `startracker.py`

实现星敏感器姿态观测。

流程是：

1. 根据光轴在惯性系内筛选可见恒星
2. 只对入视场恒星做相机系变换
3. 注入方向噪声
4. 调用 Wahba 解算姿态四元数

### 2.8 `visualization.py`

只负责画图，不参与控制闭环。

覆盖内容包括：

- 姿态估计
- 控制响应
- PD / ADRC 对比
- 参数辨识
- 优化器报告
- 综合仪表板

### 2.9 `benchmark.py`

用于项目级性能测试，帮助判断：

- 单次仿真速度
- 含传感器/辨识时的额外开销
- PD 调参整体耗时

## 3. 主数据流

单次闭环仿真的主流程如下：

1. 真实动力学给出当前 `q, w`
2. 星敏感器生成姿态观测 `q_meas`
3. 陀螺生成角速度观测 `w_gyro`
4. MEKF 执行 `predict + update`
5. 根据滤波后的角速度构造 `wdot_est`
6. 若启用在线惯量辨识，则更新 `J_diag`
7. 将惯量估计同步给 ADRC 的 `b0`
8. 根据 `control_feedback_source` 选择真实状态或估计状态作为控制反馈
9. 控制器输出力矩 `u`
10. 动力学积分推进到下一步
11. 记录误差、力矩、偏置、扰动和辨识结果

如果是 PD 自动调参的轻量路径，则流程会简化为：

1. 生成初始激励
2. 用 PD 控制器直接闭环
3. 推进动力学
4. 只记录目标函数需要的时域指标

这个简化路径的目的不是替代完整仿真，而是让大量参数评估时更高效。

## 4. 关键原理

### 4.1 姿态动力学

刚体欧拉方程：

`J * wdot = u - w x (Jw) + dist`

姿态使用四元数表达，积分时通过角速度矩阵传播。

### 4.2 星敏姿态测量

星敏感器先完成视场筛选，再用 Wahba 问题求解旋转矩阵，最终转换成姿态四元数。

### 4.3 MEKF

MEKF 不直接把四元数作为加法状态来估计，而是采用：

- 名义四元数
- 小角度误差状态

这种形式比直接在四元数上做普通 EKF 更稳定，也更适合姿态问题。

### 4.4 PD 控制

PD 结构简单，调参清晰，适合作为基线方法：

- 优点：响应快、实现简单
- 缺点：对模型误差和持续扰动的鲁棒性有限

### 4.5 LADRC

LADRC 的核心是“观测总扰动并在控制律中补偿”。

在本项目里，ADRC 的主要价值体现在：

- 面对常值扰动和噪声时稳态性能更稳
- 可以把 ESO 的扰动力矩估计拿给辨识器使用

### 4.6 在线惯量辨识

当前只估计对角惯量：

- `Jxx`
- `Jyy`
- `Jzz`

这样做的原因是：

- 数值更稳定
- 与三轴控制结构更匹配
- 在线辨识更容易收敛

## 5. 当前结构的优点与局限

### 5.1 优点

- 仿真链路完整
- 控制、估计、辨识都可以插拔
- 图像输出丰富
- 已经具备性能基准入口

### 5.2 局限

- `main.py` 仍然偏大，承担了太多职责
- 文档、参数和实验配置还没有完全对象化
- 当前 ADRC 没有像 PD 那样形成统一的自动调参流程

## 6. 后续建议

如果继续优化结构，建议优先拆分：

- `simulation_runner.py`
  - 只负责单次仿真执行
- `metrics.py`
  - 只负责性能指标计算
- `experiments.py`
  - 只负责 PD / ADRC / 辨识实验组织
- `reporting.py`
  - 只负责图像、表格和 Markdown 报告输出

这样后续继续增加实验种类时，代码耦合会明显更低。
