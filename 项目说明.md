# 卫星姿态控制仿真项目说明

## 1. 项目概览

本项目实现了一个面向刚体卫星的闭环姿态控制仿真系统，包含以下能力：

- 三轴刚体姿态动力学与 RK4 积分
- 星敏感器与陀螺仪测量建模
- MEKF 姿态与陀螺偏置估计
- PD / LADRC 两类控制器
- 五种参数优化器的横向对比
- 转动惯量在线辨识
  - 方案 A：ESO + RLS
  - 方案 B：增广 MEKF

项目当前代码以“可跑通、可扩展、便于试验”为目标，适合继续做控制律调参、估计算法替换和在线辨识验证。

## 2. 当前目录结构

### 核心仿真

- `main.py`
  - 主入口。
  - 包含单次闭环仿真 `simulate_attitude_control(...)`、PD 自动调参 `compare_pd_gain_optimizers(...)`、以及 `main(profile=...)` 对比流程。
- `dynamics.py`
  - 卫星刚体动力学。
  - 支持运行中更新惯量 `update_inertia(new_J)`。
- `controller.py`
  - PD 控制器接口。
- `adrc_controller.py`
  - 三轴 LADRC 控制器。
  - 支持运行中更新 `b0`：`update_b0(new_b0)`。
- `mekf.py`
  - `MEKFBiasOnly`：6 维误差状态 `[δθ, δb]`
  - `MEKF_Augmented`：9 维误差状态 `[δθ, δb, ΔJ]`
  - `MEKF`：兼容旧演示的简化版本
- `estimators.py`
  - 对角惯量在线辨识工具。
  - 包含 `InertiaRLS`、回归矩阵构造与惯量雅可比构造。

### 传感器与数学工具

- `startracker.py`
  - 星敏感器视场筛选、噪声注入、Wahba 解算。
- `wahba.py`
  - Wahba 问题的 SVD 姿态解算。
- `core_utils.py`
  - 四元数、旋转矩阵、姿态误差与角度计算等核心数学函数。
- `utils.py`
  - 对 `core_utils.py` 的轻量封装，并补充 `rand_unit(...)`。

### 可视化与基准

- `visualization.py`
  - 通用绘图函数。
- `benchmark.py`
  - 项目级性能基准脚本，用于比较不同仿真路径的耗时。

### 配置与优化器

- `config.py`
  - 默认参数集中配置。
- `优化器/optimizers.py`
  - `GridSearch`
  - `RandomSearch`
  - `NelderMead`
  - `SimAnneal`
  - `PSO`

## 3. 主数据流

闭环主流程如下：

1. 真实动力学给出当前姿态 `q` 和角速度 `w`
2. 星敏感器输出姿态测量 `q_meas`
3. 陀螺仪输出角速度测量 `w_gyro`
4. MEKF 进行 `predict + update`
5. 若启用惯量在线辨识，则在滤波更新后估计 `J`
6. 将新的 `J` 同步给：
   - `Spacecraft.update_inertia(...)`
   - `ADRCController.update_b0(...)`
7. 控制器输出力矩 `u`
8. 动力学积分到下一步

## 4. 在线惯量辨识说明

### 4.1 方案 A：ESO + RLS

在 `estimators.py` 中实现。

欧拉方程为：

`J * wdot + w x (Jw) = u + d`

只辨识对角惯量 `J = diag(Jxx, Jyy, Jzz)` 时，可写成线性回归：

`y = Phi * theta`

- `theta = [Jxx, Jyy, Jzz]^T`
- `y = u + d_hat`
- `d_hat` 由 ADRC 的 ESO 等效力矩估计给出

同时加入两类安全措施：

- 激励不足时冻结更新，避免参数漂移
- 单步参数更新限幅，避免短时噪声造成惯量跳变

### 4.2 方案 B：增广 MEKF

在 `mekf.py` 中实现 `MEKF_Augmented`。

增广状态为：

`X = [δθ, δb, ΔJ]^T`

其中 `ΔJ` 采用随机游走模型。姿态量测仍只直接观测 `δθ`，惯量通过预测阶段建立的交叉协方差被间接修正。

## 5. 运行方式

### 5.1 主流程

运行完整对比：

```bash
python main.py
```

代码内部支持两种运行档位：

- `main(profile='full')`
  - 完整仿真与对比图输出
- `main(profile='fast')`
  - 更短仿真时间
  - 更少优化预算
  - 默认跳过大型对比图生成

### 5.2 基准测试

运行项目级性能基准：

```bash
python benchmark.py
```

该脚本会对几条典型路径做耗时统计，例如：

- `PD_no_tracker`
- `ADRC_tracker`
- `ADRC_tracker_RLS`
- `PD_optimizer_quick`

## 8. 注意事项

- 当前主仿真中的控制器仍默认使用真实姿态 `q` 做闭环，这更适合看控制极限性能。
- 如果要验证“仅靠估计姿态闭环”，需要将控制输入切换为 `mekf.q`。
- 在线惯量辨识目前只针对对角惯量，不处理惯量耦合项。
