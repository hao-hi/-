# 卫星姿态控制仿真项目说明

## 1. 项目定位

本项目是一个面向刚体卫星的姿态控制闭环仿真工程，目标不是只做单一控制器演示，而是把以下能力放在同一套实验框架里：

- 姿态动力学传播
- 星敏感器与陀螺测量建模
- MEKF 姿态与偏置估计
- PD / LADRC 控制器对比
- PD 参数自动调优
- 对角惯量在线辨识
- 图像、表格与实验报告输出

如果只想快速了解项目，建议按下面顺序阅读：

1. 本文档：看整体用途、运行方式和输出结构
2. `代码结构与原理说明.md`：看模块关系和算法主线
3. `仿真结果解释说明.md`：看图怎么读、指标怎么解释

## 2. 当前代码能力

### 2.1 闭环仿真

主入口在 `main.py`，核心函数包括：

- `simulate_attitude_control(...)`
  - 单次仿真主函数
  - 支持 `PD` 与 `ADRC`
  - 支持星敏观测开关
  - 支持在线惯量辨识开关
- `compare_pd_gain_optimizers(...)`
  - 对 `Kp, Kd` 做自动调参
  - 横向比较 5 种优化器
- `main(profile='fast'/'full')`
  - 组织“调参 -> 单次 PD 仿真 -> 单次 ADRC 仿真 -> 对比图/报告输出”的完整流程

### 2.2 在线惯量辨识

支持两条辨识路线：

- `RLS`
  - 基于 ESO 扰动力矩估计 + 线性回归 + RLS
  - 只辨识 `Jxx, Jyy, Jzz`
- `MEKF`
  - 使用增广误差状态 `[δθ, δb, ΔJ]`
  - 通过动力学残差间接修正惯量

### 2.3 图像与数据输出

程序会把输出统一写入 `output/`，按类别区分目录：

- `output/optimization`
- `output/comparison`
- `output/identification`
- `output/simulations/pd`
- `output/simulations/adrc`

每组输出通常包含：

- 核心图像
- 摘要指标表
- 时序数据 CSV
- Markdown 报告

## 3. 目录结构

### 3.1 核心仿真层

- `main.py`
  - 实验组织、仿真主循环、结果汇总、报告输出
- `dynamics.py`
  - 刚体姿态动力学与 RK4 积分
- `controller.py`
  - PD 控制器

  - `MEKFBiasOnly`
  - `MEKF_Augmented`

- `startracker.py`
  - 星敏感器观测模型与 Wahba 姿态解算

### 3.3 数学与工具层

- `core_utils.py`
  - 四元数、旋转矩阵、姿态误差与随机单位向量等核心数学函数
- `config.py`
  - 默认参数

### 3.4 输出与性能分析

- `visualization.py`
  - 所有图像生成函数
- `benchmark.py`
  - 基准测试入口
- `optimizers.py`
  - 兼容旧目录 `优化器/optimizers.py` 的导入写法
  - `GridSearch / RandomSearch / NelderMead / SimAnneal / PSO`

## 4. 运行方式

### 4.1 完整实验

```bash
python main.py
```

默认等价于：

```python
main(profile='full')
```

### 4.2 快速实验

```python
from main import main
main(profile='fast')
```

`fast` 档位会缩短总时长、减少优化预算，并跳过部分大型图输出，适合快速验证代码链路是否正常。

### 4.3 单次仿真

```python
from main import simulate_attitude_control

res = simulate_attitude_control(
    controller_type='PD',
    T=8.0,
    dt=0.03,
    show_plots=False
)
```

常用开关：

- `controller_type='PD'` 或 `'ADRC'`
- `use_star_tracker=True/False`
- `inertia_estimator_cfg=None / {'scheme': 'RLS'} / {'scheme': 'MEKF'}`
- `excitation_profile='auto'/'nominal'/'aggressive'`
- `control_feedback_source='truth'/'estimate'`
- `result_mode='full'/'metrics'`

其中：

- `result_mode='full'`
  - 返回完整时序，用于画图和导出
- `result_mode='metrics'`
  - 当前主要用于 PD 自动调参
  - 走轻量路径，只保留目标函数所需指标，明显降低优化耗时

### 4.4 性能基准

```bash
python benchmark.py
```

当前基准脚本会统计例如：

- `PD_no_tracker`
- `ADRC_tracker`
- `ADRC_tracker_RLS`
- `PD_optimizer_quick`

## 5. 近期代码整理与优化

本轮已完成的重点优化如下：

- 将 PD 调参路径改为轻量指标模式，避免每次目标函数评估都走完整传感器/MEKF 链路
- 提炼了统一的初始激励生成逻辑，减少不同仿真路径的重复代码
- 将主循环中的历史数组访问改为局部引用，降低热路径字典开销
- 把仿真中的随机量改成批量预生成，减少循环内 Python 调用

在当前机器上的一次基准结果中，`PD_optimizer_quick` 的耗时已从优化前约 100 秒量级降到约 26 秒量级。

说明：

- 这个数字是当前环境下的一次实测参考
- 具体耗时会受 CPU、NumPy/Numba 缓存状态和 Python 环境影响

## 6. 使用时要注意的地方

- 主仿真默认仍使用真实姿态 `q` 参与控制闭环，更适合评估控制器极限性能。
- 如果要测试“估计姿态闭环”，可直接设置 `control_feedback_source='estimate'`，让控制器使用 `mekf.q` 与去偏后的角速度估计闭环。
- 在线惯量辨识当前只支持对角惯量，不处理惯量耦合项。
- `main.py` 目前仍承担了较多组织职责，后续如果继续扩展实验场景，建议拆分模块。

## 7. 推荐后续工作
