## 摘要

卫星姿态控制问题同时受到模型不确定性、外部扰动、测量噪声、执行器约束以及参数偏差等多种因素影响。若仅针对某一局部模块开展研究，往往难以反映真实闭环系统在“建模、测量、估计、控制与参数适应”多环节耦合条件下的实际表现。为此，本文围绕刚体卫星姿态稳定控制问题，设计并实现了一套面向研究与实验分析的综合仿真平台。该平台采用标量在前的四元数描述姿态，利用欧拉方程构建三轴刚体姿态动力学模型，并在仿真中引入常值扰动、随机扰动和执行机构力矩饱和约束。在测量链路方面，平台建立了包含星表生成、视场筛选、方向噪声注入和 Wahba 姿态解算的星敏感器模型，同时建立了含偏置随机游走的陀螺仪测量模型。

在状态估计方面，本文实现了基于乘法扩展卡尔曼滤波器的姿态与陀螺偏置联合估计方法，并进一步实现了用于对角惯量联合修正的增广状态 MEKF。针对控制与参数不确定问题，平台集成了基于四元数误差的 PD 控制器和三轴并行 LADRC 控制器，并设计了面向 PD 参数自动整定的多优化器横向比较流程。在参数在线辨识方面，本文实现了基于 ESO 扰动力矩估计与递推最小二乘的对角惯量辨识方法，以及基于增广状态滤波的惯量联合估计方法。为了提高调参实验效率，平台还构造了面向目标函数评估的轻量化指标仿真路径，在保证评价口径与完整仿真一致的前提下降低了重复评估开销。

当前平台已经能够完成姿态动力学传播、传感器测量生成、姿态与偏置估计、控制器性能比较、PD 参数优化、ADRC 带宽整定、在线惯量辨识以及图表和 Markdown 报告自动输出等功能。在当前统一的 `fast` 档代表性实验中，前置辨识阶段综合 RLS 与增广状态 MEKF 的结果，并基于短时闭环验证选择融合名义惯量 `[1.5569, 1.3437, 0.9861] kg·m²` 作为后续 ADRC 配置依据；PD 调参阶段以 Nelder-Mead 获得最优平均得分，对应参数为 `Kp=1.4466`、`Kd=1.5739`；控制器正式比较结果显示，整定后的 LADRC 在该场景下取得了 `1.9600 s` 的调节时间和 `0.0871 deg` 的末端姿态误差，优于 PD 控制器的 `6.2800 s` 和 `1.9474 deg`，但 PD 在控制能耗、峰值力矩与饱和行为方面更为保守。在线惯量辨识结果表明，两类辨识方法均可实现对角惯量修正，其中当前实验下增广状态 MEKF 的最终惯量误差范数更小，而融合模型在闭环验证中给出了更稳妥的综合表现。此外，本文还对项目进行了面向研究效率的代码优化，使 `ADRC_tracker_RLS` 基准耗时由 `0.0804 s` 降至 `0.0252 s`，`PD_optimizer_quick` 由 `3.3130 s` 降至 `0.2675 s`。上述结果说明，该平台已具备开展姿态控制比较研究和参数不确定性分析的实际能力，可作为后续正式论文扩展、多场景统计分析和算法升级研究的统一基础。

## 1. 引言

### 1.1 研究背景与意义

卫星姿态控制系统是航天器实现稳定指向、对地观测、通信链路维持、载荷定向以及在轨服务的重要基础。随着空间任务逐步向高精度、长时段和复杂环境演化，姿态控制问题不再只是理想模型下的稳定化问题，而是一个同时受动力学建模误差、未知外部扰动、传感器噪声、执行器约束以及参数时变特性影响的综合性问题。尤其是在高精度遥感、窄波束通信、星间链路保持和长期稳定观测等任务中，较小的姿态偏差也可能转化为明显的任务性能损失，因此如何在不确定条件下兼顾动态响应速度、稳态误差、控制代价和鲁棒性，始终是姿态控制研究中的关键问题。

从工程实现角度看，姿态控制系统的表现并不单纯由控制律本身决定。即使控制器设计合理，若姿态测量链路存在偏差、滤波器对陀螺偏置估计不准确，或者动力学模型中的惯量参数与真实对象存在偏差，闭环系统性能仍然会明显下降。同样，参数辨识方法的效果也并不是孤立存在的，其更新是否稳定、辨识是否收敛，往往取决于控制器所提供的激励水平、观测器的扰动估计质量以及测量链路的噪声条件。因此，建立一套能够同时刻画动力学、测量、估计、控制和参数辨识耦合关系的综合仿真平台，对于算法比较、性能分析和后续扩展研究都具有现实意义。

### 1.2 研究问题与挑战

本文关注的核心问题并非“某一个控制器是否优于另一个控制器”，而是如何构建一条完整、可复现、可扩展的研究链路，使得控制器比较、参数优化和在线辨识等研究任务能够在统一平台上开展。围绕这一目标，当前工作主要面临以下几方面挑战。

首先，姿态系统具有显著的非线性和几何约束特性。若直接使用欧拉角建模，容易引入奇异性问题；若直接将四元数作为普通加法状态处理，又会带来单位模约束维护困难和线性化误差放大等问题。因此，在建模与滤波环节，需要同时兼顾姿态表示的一致性、数值传播稳定性和误差状态的合理定义。

其次，测量链路本身具有多源、异质和耦合的特点。星敏感器能够提供姿态观测，但受到视场限制、可见星数变化和测量噪声影响；陀螺仪能够提供高频角速度测量，但存在偏置和噪声积累问题。如何在统一仿真框架中合理刻画这两类传感器，并通过滤波器有效融合，是姿态闭环仿真能够成立的关键前提。

再次，控制器设计与参数不确定性之间存在天然矛盾。传统 PD 控制器结构简单、调参清晰，但对模型误差和持续扰动的鲁棒性有限；LADRC 通过观测总扰动并补偿，可在弱模型条件下表现出更强的抗扰能力，但其带宽参数、内部模型和实际力矩约束之间的匹配同样需要细致处理。此外，当惯量参数存在偏差时，控制器内部模型和真实对象之间的失配会进一步放大控制性能差异。

最后，参数辨识属于慢变量估计问题，其有效更新对激励条件和噪声抑制能力高度敏感。对角惯量并不能被姿态量测直接观测，而需要借助角速度、角加速度和控制力矩之间的动力学关系间接修正。若系统激励不足，辨识器容易冻结；若差分量测噪声较大，则辨识结果又容易抖动甚至漂移。因此，在线辨识方法的实现不仅是数学公式推导问题，也涉及滤波、平滑、更新门限、创新裁剪和协方差约束等多种工程化稳定化措施。

围绕上述问题，本文在现有代码项目基础上完成了以下工作。

第二，构建了包含星敏感器与陀螺仪的测量仿真链路。星敏感器部分实现了星表生成、视场筛选、方向噪声注入和 Wahba 姿态解算，陀螺仪部分实现了含偏置随机游走的角速度测量模型。该测量链路能够为姿态估计与闭环实验提供更接近实际问题的观测条件。

第五，构建了面向调参任务的自动优化流程。针对 PD 控制器，平台集成了网格搜索、随机搜索、Nelder-Mead、模拟退火和粒子群优化等多种优化器，并采用多随机种子评估和风险敏感目标函数降低对单一样本工况的过拟合；针对 LADRC，平台实现了基于已辨识惯量的控制带宽与观测器带宽轻量整定流程。

第六，建立了统一的实验输出机制。平台能够自动保存单次仿真图像、控制器对比图、在线辨识仪表板、优化器报告图、摘要指标表、时序 CSV 与 Markdown 汇总报告，从而使平台同时具备研究记录、阶段汇报和正式论文写作的支撑能力。

## 2. 相关工作

### 2.1 姿态表示与姿态估计相关研究

姿态表示是航天器姿态控制与估计问题的基础。欧拉角表示直观，但在大机动条件下存在奇异性；方向余弦矩阵虽然表达完整，但参数维度较高且存在正交约束维护问题；四元数表示则兼顾了全局无奇异性和计算效率，因此在航天器姿态控制与姿态确定问题中得到广泛应用。对于采用四元数表示的姿态系统，姿态估计方法一般包括传统扩展卡尔曼滤波、无迹卡尔曼滤波以及误差状态滤波等多种形式。其中，乘法扩展卡尔曼滤波器通过“名义四元数 + 小角度误差状态”的建模方式，在保持单位四元数约束和减小线性化误差之间取得较好平衡，已成为航天器姿态估计中的经典方法。

与直接在四元数空间中进行加法更新相比，MEKF 更适合处理姿态系统这一具有特殊几何结构的估计问题。其核心思路是：将四元数本体用于名义姿态传播，而将滤波误差以小角度向量形式定义，从而避免在滤波更新中直接破坏四元数单位模约束。对本文而言，采用 MEKF 的意义不仅在于获得姿态估计值，更在于为后续偏置估计、角速度修正以及动力学参数联合估计提供统一误差状态框架。

### 2.2 姿态控制相关研究

在航天器姿态控制领域，PD 类控制方法由于结构简单、参数物理意义明确、实现代价较低，一直是最常见的基线方法之一。对于姿态稳定控制问题，若系统模型较为准确、执行器能力充足且扰动相对可控，PD 控制器通常能够给出较快的动态响应，因此在工程上具有较高实用性。然而，PD 控制本质上并不主动估计和补偿未知扰动，面对持续外扰、模型失配和参数偏差时，其稳态性能和鲁棒性往往受到限制。

自抗扰控制为弱模型条件下的控制问题提供了另一种思路。其核心思想是将系统内外不确定因素统一视为总扰动，并通过扩张状态观测器进行在线估计，再在反馈律中进行补偿。LADRC 作为自抗扰控制的线性化实现形式，具有结构相对规整、带宽参数化清晰、与传统控制器便于比较等特点。对于本文平台而言，引入 LADRC 的主要目的，不是宣称其在所有场景下都优于 PD，而是希望在统一仿真框架下研究“快速响应基线控制”与“带扰动估计补偿控制”两类思路在动态性能、稳态误差和控制代价上的差异。

### 2.3 在线参数辨识相关研究

在线参数辨识问题广泛存在于飞行器、机器人和机电伺服系统中。对于结构已知、参数未知的动力学模型，递推最小二乘方法因其实现直观、计算代价较低、工程解释性强而被广泛使用；对于希望在统一概率框架中同时估计系统状态与参数的场景，则常采用增广状态滤波思路，将未知参数作为缓慢变化状态并入滤波器统一估计。对于航天器姿态问题而言，惯量参数属于动力学核心参数，其偏差会直接影响角速度传播、控制器内部模型和扰动估计精度，因此对惯量参数开展在线辨识具有明确的工程动机。

然而，惯量参数的在线辨识并不容易。由于姿态传感器并不直接测量惯量参数，参数估计只能通过角速度、角加速度、力矩输入和扰动之间的关系间接开展。若系统激励水平不足，参数就难以被有效辨识；若角加速度由噪声较大的离散差分获得，辨识结果又会受到严重污染。也正因如此，本文在实现基于 RLS 的对角惯量辨识时，引入了激励门限、滑动窗口、创新裁剪、单步参数限幅与协方差投影等稳定化措施；在实现增广状态 MEKF 时，则将惯量参数视为慢变量，并通过动力学伪量测进一步修正惯量估计。

### 2.4 综合仿真平台化研究

从研究方法论角度看，单独验证某个控制器、单独验证某个滤波器或单独验证某个辨识器，虽然可以说明局部算法的可行性，但往往无法反映完整闭环中“测量误差影响控制、控制激励影响辨识、辨识结果反过来影响控制器内部模型”的耦合关系。因此，建立一套可重复调用、具备统一输出和统一评价口径的综合仿真平台，是系统化研究姿态控制问题的重要基础。

本文的工作定位正是在这一层面展开。平台既不追求一次性覆盖所有先进控制算法，也不试图在当前版本中完成所有工程复杂性建模，而是围绕姿态动力学、测量、估计、控制、调参与辨识等核心环节形成一条闭环实验链路。通过统一的数据结构、统一的实验组织入口和统一的输出目录，平台能够为后续论文写作、课程汇报和进一步算法扩展提供稳定基础。

## 3. 系统模型与问题描述

### 3.1 四元数姿态运动学模型

本文采用标量在前的单位四元数表示卫星姿态，即

`q = [q0, q1, q2, q3]^T`

其中 `q0` 为标量部，`[q1, q2, q3]^T` 为向量部。设卫星在本体系下的角速度向量为

`w = [wx, wy, wz]^T`

则姿态运动学方程可写为

`qdot = 0.5 * Omega(w) * q`

其中

`Omega(w) = [[0, -wx, -wy, -wz], [wx, 0, wz, -wy], [wy, -wz, 0, wx], [wz, wy, -wx, 0]]`

四元数表示在大角度机动条件下不存在欧拉角奇异性，因此适合作为姿态控制与姿态估计问题的统一状态变量。为了保证数值传播过程中单位模约束不被积分误差破坏，平台在每次姿态传播后都对四元数进行归一化处理。当前代码实现中，动力学积分采用四阶 Runge-Kutta 方法推进姿态和角速度，以兼顾数值稳定性与仿真精度。

在本文平台中，四元数不仅用于描述真实姿态，也被用于构造控制误差和姿态测量残差。设期望姿态为 `qd`，当前姿态为 `q`，则误差四元数定义为

`q_e = qd^(-1) ⊗ q`

当前代码会在误差四元数标量部为负时对其整体取反，使误差始终沿四元数双覆盖中的短弧方向表达。这样做的原因在于，单位四元数 `q` 与 `-q` 对应同一物理姿态，但若不统一符号，基于向量部的小角度误差近似和控制律会出现不必要的数值跳变。也正因如此，`core_utils.py` 中对四元数乘法、共轭、误差计算和旋转矩阵变换做了统一封装，使控制器、滤波器和传感器模型共享同一套姿态数学基础。

### 3.2 刚体姿态动力学模型

在动力学建模中，本文将卫星简化为刚体，姿态转动动力学满足欧拉方程

`J * wdot = u - w x (Jw) + d`

其中 `J` 为转动惯量矩阵，`u` 为控制力矩，`d` 为外部扰动力矩。若 `J` 可逆，则有

`wdot = J^(-1) * (u - w x (Jw) + d)`

为了与当前控制器结构和在线辨识实现保持一致，平台主要考虑对角惯量情形，即

`J = diag(Jxx, Jyy, Jzz)`

这一假设一方面符合当前平台的研究目标，另一方面也使得在线辨识问题能够在数值稳定性和工程可解释性之间取得较合理平衡。对于对角惯量模型，角速度导数可以在各轴上写为显式形式，从而在热路径中减少一般矩阵乘法和求逆带来的额外开销。

对于 `J = diag(Jx, Jy, Jz)` 的情形，上式可进一步按轴展开为

`Jx * wdot_x = u_x - (Jz - Jy) * wy * wz + d_x`

`Jy * wdot_y = u_y - (Jx - Jz) * wx * wz + d_y`

`Jz * wdot_z = u_z - (Jy - Jx) * wx * wy + d_z`

这组显式方程一方面体现了刚体姿态动力学中的角速度耦合项，另一方面也为后续对角惯量在线辨识提供了直接的回归基础。当前代码中的 `Spacecraft` 类正是利用这组显式关系，为对角惯量情况实现了快速分支。

在数值求解层面，平台采用经典四阶 Runge-Kutta 方法同时推进 `q` 与 `w`。其基本思想是：在每一时间步内分别计算状态导数在四个采样点上的近似值，并按 `1:2:2:1` 组合得到下一时刻状态。相较于简单欧拉法，这种处理在角速度较大、扰动持续作用或控制输入变化较快时更能抑制传播误差。

### 3.3 扰动与执行器约束

为使仿真对象具有代表性，本文在姿态动力学中同时引入常值扰动和小幅随机扰动。常值扰动用于近似反映长期存在的未建模偏置力矩，如重力梯度、气动不平衡、结构装配误差或残余执行器偏差；随机扰动则用于模拟较高频的环境不确定项。平台中的扰动力矩模型写为

`d = d_const + d_noise`

其中 `d_const` 为常值项，`d_noise` 为零均值随机噪声项。控制器输出进一步经过执行器限幅：

`u_sat = clip(u, -u_max, u_max)`

力矩饱和不仅会影响闭环动态响应，也会直接改变实际施加到系统上的输入，因此在本文的 LADRC 设计中，ESO 明确使用上一时刻实际施加的饱和后力矩作为观测器输入，以避免观测器在饱和条件下因“期望力矩”和“实际力矩”不一致而产生等效积分风up。

需要进一步说明的是，扰动与饱和并不只影响控制器表现，还会同时影响在线辨识质量。若系统长时间处于小误差稳态区间，角速度与角加速度变化幅度都会下降，辨识器能够利用的有效激励随之减弱；若控制输入频繁进入饱和，则“控制器输出”和“实际施加力矩”之间出现明显偏差，若观测器和辨识器仍使用未饱和力矩建模，就会进一步放大模型不一致误差。当前平台在控制、观测和辨识三个环节中都尽量引用饱和后的实际施加力矩，目的正是降低这一类误差传播。

### 3.4 星敏感器与陀螺仪测量模型

#### 3.4.1 星敏感器模型

星敏感器观测链路由星表生成、视场筛选、相机系方向向量构造、方向噪声注入和 Wahba 姿态解算组成。设惯性系参考星向量为 `r_i`，机体系观测向量为 `v_i`，则姿态解算问题可表述为 Wahba 问题

`min_R sum(a_i * ||v_i - R * r_i||^2)`

其中 `R` 为姿态旋转矩阵，`a_i` 为权重。当前实现采用基于奇异值分解的 Wahba 解算方式，从观测方向向量与参考方向向量恢复旋转矩阵，并进一步转换为姿态测量四元数。为了提高效率，平台首先根据相机光轴在惯性系中的方向进行可见星筛选，仅对落入视场的恒星做相机系投影和噪声注入。当可见星数不足时，平台采用退化观测策略，用带小角度噪声的真实姿态替代观测结果，以保证滤波链路不因观测缺失而完全失效。

从流程角度看，星敏感器单次观测可分为以下步骤：首先根据当前姿态确定相机光轴在惯性系中的方向；随后与随机生成的星表做点乘筛选，保留落入视场的恒星；然后将可见星方向向量映射到相机系并叠加方向噪声；接着选取靠近视场中心的代表性恒星参与 Wahba 解算；最后由解算得到的旋转矩阵恢复姿态四元数。该流程保留了“可见星数变化”这一实际特征，因此比直接在姿态四元数上叠加噪声更接近真实观测链路。

#### 3.4.2 陀螺仪测量模型

陀螺仪测量模型写为

`w_gyro = w + b + n`

其中 `b` 为零偏项，`n` 为高斯白噪声。偏置项进一步采用随机游走模型：

`b(k+1) = b(k) + eta_b`

其中 `eta_b` 为小幅随机扰动。该模型能够较好地刻画陀螺在较长时间仿真中的偏置漂移现象，也为 MEKF 中的偏置状态估计提供了现实动机。需要指出的是，当前平台中的陀螺模型旨在为控制与估计研究提供代表性含噪观测，而不是建立某一型号陀螺器件的精确物理误差模型。

在仿真实现中，陀螺测量不仅用于滤波器预测，还会进一步影响角加速度估计。平台先利用 MEKF 输出的偏置估计修正陀螺测量，得到滤波角速度，再通过当前与上一拍滤波角速度之差构造离散角加速度估计，并对其进行指数平滑。也就是说，陀螺偏置估计质量不仅决定姿态估计精度，还会进一步影响在线惯量辨识所依赖的 `wdot` 构造质量。

### 3.5 问题定义

基于上述建模，本文所研究的问题可以概括为：在已知期望姿态、存在外部扰动、传感器噪声、执行器饱和和惯量参数不确定性的条件下，设计姿态估计方法、控制策略和在线辨识方法，使系统能够在统一仿真平台上完成以下任务：

1. 稳定估计姿态与陀螺偏置；
2. 通过 PD 与 LADRC 控制实现姿态稳定控制；
3. 在统一评价指标下比较不同控制策略的闭环性能；
4. 对对角惯量参数进行在线修正；
5. 通过自动调参和图表输出支撑后续实验分析与论文写作。

需要特别说明的是，当前项目默认在控制闭环中使用真实姿态 `q` 与真实角速度 `w` 计算控制力矩，旨在首先评估控制器在理想状态获取条件下的极限控制性能；若研究目标转向“估计姿态闭环”，则应将控制器输入切换为滤波器输出 `mekf.q` 和对应估计量。

## 4. 姿态估计与在线参数辨识方法

### 4.1 MEKF 姿态与偏置估计

对于四元数姿态系统，若直接使用普通加法型 EKF 进行状态更新，会在保持单位四元数约束、处理姿态误差线性化以及控制误差传播范围等方面引入额外困难。基于这一原因，本文采用乘法扩展卡尔曼滤波框架，将姿态估计分解为名义四元数和误差状态两部分。对于仅估计姿态和陀螺偏置的滤波器，误差状态定义为

`x = [δθ, δb]^T`

其中 `δθ` 为小角度姿态误差，`δb` 为陀螺偏置误差。

在预测阶段，名义姿态根据去偏置后的角速度测量进行传播：

`q_hat(k+1|k) = q_hat(k|k) ⊗ dq(omega_meas - b_hat, dt)`

同时，误差协方差利用离散线性化误差模型传播：

`P(k+1|k) = Phi * P(k|k) * Phi^T + Q`

其中 `Phi` 为姿态误差和偏置误差的离散状态转移矩阵，`Q` 为过程噪声协方差。当前实现中，误差状态传播采用简化模型 `δθ_{k+1} ≈ δθ_k - δb_k * dt`、`δb_{k+1} ≈ δb_k`，这一建模在采样步长较小且偏置变化较慢的条件下具有较好适用性。

在更新阶段，滤波器首先利用姿态测量四元数与名义姿态四元数构造误差四元数，并进一步提取小角度姿态残差。由于姿态测量只直接观测小角度姿态误差，因此量测矩阵仅对 `δθ` 部分敏感。卡尔曼增益由

`K = P * H^T * (H * P * H^T + R)^(-1)`

给出，其中 `R` 为姿态测量噪声协方差。得到误差状态修正量后，滤波器一方面更新偏置估计，另一方面构造小角度修正四元数，将其左乘到名义姿态上完成乘法更新。该过程使姿态估计既保留了四元数传播的几何一致性，又保留了误差状态滤波的数值稳定性。

从算法流程上看，`MEKFBiasOnly` 在每一采样时刻可概括为：首先利用当前陀螺测量减去偏置估计传播名义姿态；其次根据离散误差模型传播协方差；然后由姿态测量与名义姿态构造小角度姿态残差；接着计算卡尔曼增益并更新误差状态；最后用误差状态修正偏置估计和名义姿态。当前实现还对协方差矩阵执行对称化和有界投影处理，以防止长时间递推中出现数值非对称或协方差退化现象。

### 4.2 增广状态 MEKF

为了在统一状态估计框架中引入惯量参数在线修正，本文在 `MEKFBiasOnly` 基础上实现了增广状态 MEKF。其误差状态定义为

`X = [δθ, δb, ΔJ]^T`

其中 `ΔJ` 表示对角惯量偏差。当前实现只考虑 `Jxx`、`Jyy` 和 `Jzz` 三个对角惯量参数，因此 `ΔJ` 为三维向量。

在预测阶段，名义姿态与偏置传播方式与 `MEKFBiasOnly` 基本一致，不同之处在于：平台进一步利用上一时刻实际施加力矩和当前估计惯量，构造角加速度模型对惯量参数的雅可比矩阵 `G_J = ∂wdot / ∂J`，并将惯量误差对姿态误差传播的影响近似折算进离散状态转移矩阵的右上块，即

`δθ_{k+1} ≈ δθ_k - δb_k * dt + 0.5 * dt^2 * G_J * ΔJ`

这一本质上是一种工程上可实现的近似处理，使惯量偏差虽不被姿态量测直接观测，但能够通过交叉协方差逐步影响姿态误差更新。

在姿态量测更新阶段，增广 MEKF 仍然只直接使用姿态残差修正 `δθ`，而 `ΔJ` 的更新来自交叉协方差所传递的信息。为增强参数估计效果，当前实现进一步引入了基于角加速度伪量测的 `update_dynamics` 过程。该过程利用滤波器内部保存的上一拍实际施加力矩、当前角速度修正值和角加速度估计，构造动力学伪量测残差，并通过额外的量测更新直接修正惯量状态。为了抑制噪声驱动和发散风险，该过程还包含激励门限判断、创新裁剪、单步惯量更新限幅和协方差有界投影等稳定化处理。

换言之，增广状态 MEKF 的参数修正并不是完全依赖姿态量测本身，而是依赖两条信息通道共同作用：一条是姿态量测通过交叉协方差间接修正惯量，另一条是角加速度伪量测直接修正惯量。前者更强调统一滤波框架的一致性，后者则增强了参数状态的可观测性。当前代码将二者结合，实质上是在理论统一性与工程可用性之间做折中。

### 4.3 基于 ESO + RLS 的对角惯量在线辨识

在对角惯量假设下，欧拉方程能够写为线性回归形式。设

- `theta = [Jxx, Jyy, Jzz]^T`
- `y = u + d_hat`

其中 `d_hat` 为由 LADRC 内部 ESO 输出换算得到的等效扰动力矩估计，则有

`y = Phi(omega, wdot) * theta`

回归矩阵 `Phi` 由当前角速度和角加速度构造，分别反映对角惯量在各轴动力学中的作用关系。其显式形式可写为

`Phi = [[wdot_x, -wy*wz,  wy*wz],`

`       [wx*wz,  wdot_y, -wx*wz],`

`       [-wx*wy, wx*wy,  wdot_z]]`

与离线最小二乘不同，在线辨识需要在滚动时间窗内持续更新参数估计，因此本文采用递推最小二乘框架进行实时辨识。其核心更新为

`K_rls = P * Phi^T * (lambda * I + Phi * P * Phi^T)^(-1)`

`theta(k+1) = theta(k) + K_rls * (y - Phi * theta(k))`

`P(k+1) = (I - K_rls * Phi) * P / lambda`

其中 `lambda` 为遗忘因子，`P` 为参数协方差矩阵。

需要指出的是，理论上的 RLS 更新在实际实现中并不能直接照搬。由于角加速度量通常由离散差分构造，噪声放大现象会非常明显；若在激励不足或稳态噪声主导时仍强行更新，惯量估计结果极易漂移。为此，本文在实现中采用了一系列工程化稳定化策略：首先，对角加速度估计与扰动力矩估计分别做指数平滑，降低高频抖动；其次，通过滑动窗口堆叠回归矩阵并监视奇异值，判断当前激励水平是否足以支撑参数更新；再次，对创新量和单步参数修正量进行裁剪与限幅，避免单次异常样本导致估计突变；最后，对参数协方差做上下界投影，防止长时间冻结或噪声驱动造成协方差风化。

进一步地，当前 RLS 实现并不是在每个采样点无条件更新，而是先判断角加速度激励、等效力矩激励、回归矩阵范数以及创新量是否同时满足条件。只有当这些条件成立时，参数更新才真正激活。这一设计直接体现了“辨识必须建立在有效激励之上”的工程原则，也解释了后文实验中为什么 RLS 的更新占比通常低于增广状态 MEKF。

### 4.4 两类惯量辨识方法的比较分析

从方法机理上看，基于 ESO + RLS 的辨识方法属于显式回归辨识。它的优点在于结构清晰、工程解释性强、调试逻辑相对明确，尤其适合分析“激励是否足够”“创新是否合理”“参数是否冻结”等工程问题。其局限在于对角加速度估计质量高度敏感，当系统处于稳态或激励不足阶段时，参数更新容易停滞；若噪声抑制策略设置不当，则又容易出现更新抖动。

增广状态 MEKF 则属于状态与参数联合估计框架。其优点在于理论上能够在统一滤波模型内同时处理姿态、偏置与惯量，并通过交叉协方差自然地实现状态和参数之间的信息传递。当前实现进一步利用角加速度伪量测增强了参数修正能力。该方法的局限在于参数更多、耦合更强，对过程噪声、量测噪声、更新增益和激励水平都更为敏感，调参难度通常高于显式回归方法。

因此，本文同时保留两类辨识路径，目的并不是给出“一种方法绝对优于另一种方法”的结论，而是希望在统一平台上比较两类思路在更新频率、误差收敛、稳定性和控制器模型匹配方面的差异，为后续更系统的实验研究提供基础。

## 5. 控制器设计与参数优化

### 5.1 基于四元数误差的 PD 控制器

PD 控制器是本文的基线控制器。设期望姿态四元数为 `qd`，当前姿态四元数为 `q`，则姿态误差四元数记为

`q_e = qd^(-1) ⊗ q`

设其向量部为 `q_e,v`，则 PD 控制律写为

`u = -Kp * q_e,v - Kd * w`

其中 `Kp` 和 `Kd` 分别为比例增益和微分增益，`w` 为当前角速度。该控制律的物理意义较为直观：比例项用于驱动姿态误差收敛，微分项用于提供角速度阻尼。由于 PD 控制结构简单、参数数量少、调参逻辑清晰，本文将其作为控制器比较和自动调参研究的基准对象。

在当前实现中，PD 控制器本身只承担控制律计算，不重复实现四元数误差计算和姿态数学基础函数。这样一方面保证了控制器模块与数学工具层之间的边界清晰，另一方面也使得不同控制器可以共享统一的姿态误差定义，从而提高比较实验的一致性。

从控制机理上看，该控制律可理解为典型的“姿态弹簧项 + 角速度阻尼项”结构：`Kp` 越大，控制器对姿态误差的回正作用越强，通常能够缩短调节时间，但同时也可能带来更大的超调和更高的峰值力矩；`Kd` 越大，系统阻尼越强，能够抑制振荡和降低超调，但若阻尼设置过高，也可能牺牲响应速度。正因为二者之间存在明显耦合，仅依赖人工经验往往难以同时兼顾快速性、平稳性和控制开销，这也是后文引入自动调参框架的重要动机。

### 5.2 线性自抗扰控制器

为研究弱模型条件下的扰动抑制能力，本文引入三轴并行 LADRC 控制器。当前实现的 LADRC 由三部分组成：跟踪微分器 TD、扩张状态观测器 ESO 和状态误差反馈律。

TD 的作用是对参考姿态进行平滑调节，避免参考信号变化过于突兀时直接将大阶跃误差传递给控制器内部观测环节。对于姿态系统，TD 并不是简单对四元数做线性插值，而是通过当前平滑参考姿态与目标姿态之间的小角度误差构造参考角速度更新项，从而在四元数空间中实现相对平滑的参考过渡。

ESO 用于估计误差状态及总扰动。当前实现中，其状态为 `z1`、`z2` 和 `z3`，分别对应误差、误差导数以及总扰动估计项。由于姿态控制系统中实际施加的力矩可能受到饱和影响，ESO 的输入并不是理想命令力矩，而是上一拍真实施加到系统上的饱和后力矩。这一处理能够减小控制器内部模型与真实对象之间的不一致性，增强观测器的工程稳定性。

在反馈律中，控制器使用姿态误差和角速度测量构造控制输出。当前实现采用基于带宽的参数化方式，由控制带宽 `omega_c` 决定反馈增益，由观测器带宽 `omega_o` 决定 ESO 增益，并通过

`u = (Kp * e - Kd * w_gyro - z3) / b0`

生成控制命令。其中 `b0` 是对控制通道增益的估计，当前平台中将其与惯量倒数近似对应，并支持在在线辨识后根据最新惯量估计实时更新。这使得控制器与参数辨识器之间形成了清晰的联动关系。

在参数映射上，当前实现采用典型二阶闭环与三阶 ESO 的带宽参数化思想，将控制器增益和观测器增益统一写为 `omega_c` 与 `omega_o` 的函数，即

`Kp = omega_c^2`

`Kd = 2 * omega_c`

`beta1 = 3 * omega_o`

`beta2 = 3 * omega_o^2`

`beta3 = omega_o^3`

这种写法的好处在于：设计者不必分别在多个增益之间逐一试错，而是只需围绕“控制响应希望多快”和“扰动观测希望多快”这两个物理含义更直接的带宽参数开展整定。需要指出的是，观测器带宽并非越大越好。若 `omega_o` 远高于采样频率所能支撑的范围，ESO 会对测量噪声和离散化误差过度敏感。当前代码在带宽候选筛选时显式考虑了 `omega_o * dt` 的数值稳定性约束，以避免离散实现中出现高增益放大噪声的问题。

### 5.3 控制性能评价指标

为了在不同控制器和不同参数配置之间建立统一比较口径，本文选取以下指标作为主要性能评价依据：

1. `settle_time`：反映系统进入稳定小误差区间的速度；
2. `overshoot`：反映过渡过程中的峰值偏离程度；
3. `final_error`：反映末端稳态误差水平；
4. `effort`：反映控制力矩幅值积分意义下的控制代价；
5. `sat_ratio`：反映执行器接近或进入饱和区间的频率；
6. `IAE`、`ITAE`：用于从误差积分角度描述全程性能；
7. `ISU`：用于从控制输入平方积分角度描述控制开销。

这些指标分别刻画了动态响应速度、过渡过程平稳性、稳态精度、控制消耗以及执行器安全边界利用程度。通过同时考察这些指标，可以避免单纯追求“收敛快”或单纯追求“误差小”而忽略其余性能维度。

### 5.4 多优化器驱动的 PD 自动调参

经验调参虽然直观，但往往带有较强主观性，也难以支撑方法比较研究。为此，本文针对 PD 控制器建立了自动调参框架，对 `Kp` 和 `Kd` 在给定搜索边界内进行统一目标函数优化。当前平台支持的优化器包括网格搜索、随机搜索、Nelder-Mead、模拟退火和粒子群优化。

在目标函数设计上，本文并未简单线性叠加原始指标，而是对调节时间、超调量、稳态误差、控制代价和饱和比例等指标进行适度非线性压缩，并在此基础上引入多随机种子评估、均值与方差惩罚以及尾部风险惩罚，以构造相对稳健的风险敏感目标函数。这样做的主要动机是：避免优化器过拟合某一个随机噪声序列，提升所得参数在代表性场景下的可重复性。

为了保证不同优化器之间的比较公平性，平台在预算层面对不同算法进行了大致对齐，同时统一记录每种优化器的代表参数、平均得分、运行间波动和 best-so-far 收敛历史。最终输出不仅包含排序表，也包含基于已评估点构造的参数景观图和优化综合仪表板，为论文分析提供更丰富依据。

从执行流程上看，PD 自动调参可以概括为以下步骤：首先固定真实惯量、扰动设置、初始姿态采样规则和优化搜索边界；其次由统一随机种子生成一组代表性评估场景，保证不同优化器面对的目标函数环境尽可能一致；然后各优化器在各自预算内提出候选 `(Kp, Kd)` 组合，并调用轻量化指标评估路径计算综合得分；接着平台缓存已评估参数点及其指标结果，避免重复计算；最后按照统一目标函数对全部候选方案排序，并选出代表性最优参数用于正式仿真验证。该流程使“调参”从经验试错转变为可复现、可比较、可归档的实验过程。

### 5.5 面向调参任务的轻量化指标评估路径

在自动调参问题中，若每次目标函数评估都执行完整的传感器仿真、MEKF 更新和全部图像构造，计算成本会迅速增长，进而限制优化器横向比较的可行性。针对这一问题，本文在平台中设计了 `result_mode='metrics'` 的轻量化评估路径。该路径保留与完整仿真一致的初始激励规则、扰动序列生成逻辑和刚体动力学传播方式，但跳过完整测量链路和图像输出，只保留目标函数所需的误差、控制和饱和指标计算。

为了进一步提高评估效率，平台还将轻量化 PD 指标仿真内核通过 Numba 加速，并将多类随机量在主循环前批量生成。需要强调的是，该轻量化路径的目标仅是提高调参效率，而不是替代完整闭环仿真。正式控制性能分析仍应以完整仿真输出为准。

### 5.6 基于辨识模型的 ADRC 带宽整定

除了 PD 自动调参之外，平台还实现了 ADRC 控制带宽和观测器带宽的轻量整定流程。当前实现以已辨识得到的名义惯量作为控制器内部模型基础，通过粗网格与细网格两阶段候选搜索，对 `omega_c` 与 `omega_o` 进行组合评估，并以 ADRC 风格的综合评分函数筛选代表参数。

这种整定方式虽然尚未形成严格理论上的全局最优保证，但它提供了一种与当前平台结构一致、工程上易于复现的带宽选择方法。更重要的是，该流程使平台能够研究“前置辨识结果如何影响控制器内部模型配置”的问题，从而形成“辨识—控制耦合研究”的闭环实验链路。换言之，ADRC 的整定并不是脱离辨识结果独立完成的，而是在“先确定名义惯量、再映射内部增益、最后验证闭环表现”的顺序下展开，这使得控制器整定过程天然带有模型匹配研究属性。

## 6. 仿真平台设计与实现

### 6.1 平台总体结构

从软件架构角度看，当前平台可分为四层：数学与物理基础层、测量与估计层、控制与优化层、实验组织与输出层。数学与物理基础层负责提供四元数运算、旋转矩阵变换、姿态积分和刚体动力学传播等基础能力；测量与估计层负责星敏感器观测、陀螺测量建模、姿态估计与惯量辨识；控制与优化层负责 PD、LADRC 和参数搜索算法；实验组织与输出层则负责仿真调度、结果汇总、图表生成和报告导出。

这种分层方式使平台既保留了相对清晰的职责边界，又能够在实验组织层灵活组合不同模块。与很多只针对单次仿真的示例脚本不同，本文平台特别强调“实验组织能力”，即在同一入口下依次完成前置辨识、控制器整定、正式仿真、结果对比和统一输出。

### 6.2 单次闭环仿真数据流

单次闭环仿真的核心流程可概括为以下步骤。首先，根据统一的初始激励规则生成初始姿态和角速度状态，并初始化动力学对象、控制器、滤波器与辨识器。其次，由真实姿态构造星敏感器姿态观测，由真实角速度与偏置随机游走生成陀螺仪测量。随后，MEKF 完成预测与更新，输出姿态和偏置估计；再利用当前与上一拍滤波角速度构造角加速度估计。若启用在线辨识，则进一步由 RLS 或增广 MEKF 更新对角惯量估计，并将该估计同步到 ADRC 的内部参数 `b0`。在此基础上，控制器输出控制力矩，经过执行器饱和后输入真实动力学模型，得到下一时刻姿态和角速度状态。

该流程的一个重要特征在于，它显式保留了控制器、观测器和辨识器之间的相互依赖关系。例如，LADRC 的 ESO 扰动力矩估计可为 RLS 辨识器提供 `d_hat`；惯量辨识结果又可反过来更新 ADRC 内部模型；而姿态估计器输出的偏置修正则会影响角速度与角加速度估计。这些耦合关系正是综合仿真平台能够支撑深入分析的关键所在。

### 6.3 完整实验工作流

在单次闭环仿真之外，本文平台还通过 `main()` 函数组织完整实验流程。该流程通常包括以下阶段：首先运行两类前置惯量辨识案例，得到 RLS 与增广 MEKF 的辨识轨迹和摘要信息；其次对候选辨识结果进行短时闭环验证，从而自动选择后续调参与 ADRC 参数配置所使用的名义惯量；然后分别执行 PD 自动调参和 ADRC 带宽整定；最后以统一真实惯量和统一运行档位分别执行 PD 与 ADRC 正式仿真，并输出控制器对比图、优化器图像、辨识仪表板和 Markdown 汇总报告。

这一实验流程体现了平台的研究导向特征：其目标不是只生成一组曲线，而是形成“前置辨识 -> 内部模型配置 -> 控制器整定 -> 正式对比 -> 结果归档”的一体化研究链路。对于正式论文写作而言，这一结构也非常有利于组织章节逻辑和图表索引。

若进一步按操作顺序展开，完整实验工作流可以归纳为七步：1）设定统一的真实惯量、运行档位和随机种子规则；2）分别运行 RLS 与增广状态 MEKF 的前置辨识案例；3）基于短时闭环验证分数选择后续使用的名义惯量模型；4）在轻量化评估路径上完成 PD 自动调参；5）基于所选名义惯量完成 ADRC 带宽整定；6）使用统一真实动力学配置执行 PD 与 ADRC 的正式闭环仿真；7）统一导出图像、CSV 和 Markdown 报告。通过这一顺序化设计，论文中的“方法章节”和“实验章节”能够共享同一条实现链路，减少“方法写一套、实验跑另一套”的脱节风险。

### 6.4 数据记录与结果输出机制

平台在仿真过程中记录的历史量包括时间序列、真实姿态、估计姿态、角速度、控制力矩、真实扰动、扰动估计、陀螺偏置真值、偏置估计、惯量估计、参数更新掩码、角加速度估计以及辨识激励指标等。这些量在实验结束后会被统一组装成结果字典，并进一步导出为 CSV 数据表和图像。

输出目录采用分层组织方式，通常包括：

- `output/optimization`：PD 参数优化结果；
- `output/optimization/adrc`：ADRC 整定结果；
- `output/simulations/pd`：PD 单次仿真图与数据；
- `output/simulations/adrc`：ADRC 单次仿真图与数据；
- `output/comparison`：PD 与 ADRC 对比结果；
- `output/identification/rls` 与 `output/identification/mekf`：两类辨识结果。

除图像和数据表外，平台还会自动生成 Markdown 汇总报告，将关键图像、对比指标、优化排序和辨识摘要整合在同一文本中。这一机制使平台不仅是“会跑仿真”的代码工程，也是一套具备研究记录与结果沉淀能力的实验系统。

从论文写作角度看，这种结果输出机制具有直接的方法学价值。结构化 CSV 便于生成表格、复核数值和绘制补充图；统一命名的图像文件便于在论文中建立图号索引；自动汇总的 Markdown 报告则可作为实验记录底稿，减少人工整理图表和指标时出现遗漏或口径不一致的问题。换言之，当前平台并非仅在“仿真完成后顺手导出若干文件”，而是将“代码输出即论文素材”作为系统设计目标之一，使研究过程、实验记录与论文撰写之间形成了更顺畅的衔接。

### 6.5 当前实现的特点与边界

当前平台在实现上还包含一些面向效率和研究便利性的工程设计。其一，PD 自动调参提供轻量化评估路径，显著降低了优化器横向比较成本；其二，随机量批量预生成和热路径局部变量缓存降低了循环中的 Python 调用开销；其三，统一的可视化函数使单次仿真、控制器比较和在线辨识输出风格保持一致；其四，当前代码已经支持通过 `control_feedback_source='truth'/'estimate'` 在真实状态闭环与估计状态闭环之间切换，从而为后续导航控制一体化研究预留了统一接口。

与此同时，也需要如实指出当前平台仍存在边界。最明显的一点是，`main.py` 目前承担了较多实验调度和结果汇总职责，后续若继续扩展更多实验场景，仍需考虑进一步拆分。另一个重要边界是：虽然平台已经支持估计状态闭环切换，但当前统一实验仍默认使用真实姿态和真实角速度进入控制器，这意味着本文实验结果更适合作为控制器在理想状态输入下的性能上界参考，而不宜直接将其解释为完整导航控制闭环的最终工程表现。

### 6.6 本轮性能优化与工程改进

为了提高平台在研究迭代中的使用效率，本文在统一实验前又完成了一轮针对热路径的工程优化。优化主要集中在三处。

第一，针对基于 ESO + RLS 的在线惯量辨识过程，将原先基于量测维度矩阵的递推求解改写为参数维度下的 `3x3` 精度矩阵求解。由于辨识参数只有三个对角惯量分量，这种写法与原有递推逻辑在数学上等价，但显著减少了窗口堆叠后矩阵求解的计算量。第二，针对 PD 自动调参过程，将“仅做排序、不生成图像”时仍补跑完整仿真的逻辑改为惰性触发，仅在需要绘制优化响应图和综合仪表板时才请求完整时序结果。第三，在 Numba 可用环境下，为核心 JIT 函数启用磁盘缓存，以降低跨进程和重复运行时的冷启动编译成本。

优化前后的基准结果如表 6.1 所示。测试环境保持一致，均在当前工作站、同一代码仓库下完成，比较对象为相同基准脚本中的代表性用例。

| 基准项 | 优化前平均耗时 / s | 优化后平均耗时 / s | 下降比例 |
| --- | --- | --- | --- |
| `PD_no_tracker` | `0.0139` | `0.0123` | `11.51%` |
| `ADRC_tracker` | `0.0202` | `0.0174` | `13.86%` |

从表 6.1 可以看出，本轮优化对基础 PD 和 ADRC 仿真的影响相对温和，但对“含在线辨识的闭环仿真”和“PD 自动调参”两类研究高频任务的提速十分明显。其中 `ADRC_tracker_RLS` 的降幅接近 `70%`，表明 RLS 更新过程原本确实是热路径瓶颈；而 `PD_optimizer_quick` 的耗时下降超过 `90%`，说明避免无效完整仿真补跑后，优化器比较流程已经更加贴近其真实研究需求。这样的工程改进虽然不改变控制与辨识算法本身，但直接提升了平台作为研究工具的可用性，使得多轮参数扫描和结果重跑更加高效。

## 7. 实验设计与结果分析

本章所有实验结果均来自当前项目统一重跑得到的一轮 `main(profile='fast')` 输出。采用这一口径的原因有两点：其一，该轮运行完整包含了前置辨识、PD 自动调参、ADRC 带宽整定以及最终控制器比较，能够保证实验链路完整；其二，所有结果来自同一批次输出，避免了不同运行轮次之间参数和图表不一致的问题。需要说明的是，`fast` 档位的目标在于以较低计算成本验证平台链路和代表性性能差异，因此本章结论应理解为“当前代表性场景下的实验结论”，而非大规模统计意义下的普遍结论。

### 7.1 实验设置

当前实验采用项目默认的 `fast` 运行档位，仿真总时长为 `8.0 s`，时间步长为 `0.04 s`，执行器力矩限幅为 `0.5 N·m`。真实转动惯量设置为

`J = diag(1.60, 1.40, 1.00) kg·m²`

控制器比较、前置辨识和整定过程均基于同一真实惯量展开。实验中启用星敏感器链路，视场角采用 `20 deg`，方向噪声标准差采用 `6e-4 rad`；外部扰动采用常值项与随机扰动叠加的形式，陀螺仪测量包含偏置随机游走和测量噪声。平台首先运行两组前置惯量辨识案例，并通过短时闭环验证在候选模型中选择更稳妥的融合名义惯量；然后执行 PD 参数优化和 ADRC 带宽整定；最后使用同一组代表参数分别运行 PD 与 ADRC 正式仿真。

表 7.1 给出了当前实验的主要设置。

| 参数项 | 取值 | 说明 |
| --- | --- | --- |
| 运行档位 | `fast` | 统一口径的代表性实验 |
| 仿真时长 `T` | `8.0 s` | 主控制对比实验时长 |
| 时间步长 `dt` | `0.04 s` | 全流程统一采样步长 |
| 力矩上限 `umax` | `0.5 N·m` | 执行器饱和边界 |
| 真实惯量 `J` | `diag(1.60, 1.40, 1.00) kg·m²` | 控制与辨识统一对象 |
| 星敏感器视场角 | `20 deg` | 姿态观测模型参数 |
| 星敏感器噪声 | `6e-4 rad` | 方向测量噪声标准差 |
| 控制反馈源 | `truth` | 当前统一实验默认真实状态闭环 |
| PD 参数来源 | Nelder-Mead 最优参数 | `Kp=1.4466`，`Kd=1.5739` |
| ADRC 参数来源 | 带宽整定最优结果 | `omega_c=2.8000`，`omega_o=3.3950` |
| 前置辨识选中方案 | FUSED | `J=[1.5569, 1.3437, 0.9861] kg·m²` |

本节还需要再次强调当前实验的解释边界。由于当前平台默认在控制闭环中使用真实姿态而非估计姿态，因此本章比较结果更接近控制器在理想状态获取条件下的性能对比，而不是完整导航控制闭环中的最终工程表现。

### 7.2 PD 参数优化结果

PD 自动调参阶段对 `Kp` 和 `Kd` 在给定边界内进行搜索，比较方法包括网格搜索、随机搜索、Nelder-Mead、模拟退火和粒子群优化。优化目标由调节时间、超调量、末端误差、控制代价和饱和比例等指标构成，并采用统一扰动和统一轻量化指标仿真路径完成评估。当前实验结果如表 7.2 所示。

| 排名 | 算法 | `Kp` | `Kd` | `Score` | `Ts / s` | `Final Error / deg` | `Effort / N·m·s` |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | NelderMead | 1.4466 | 1.5739 | 10.3924 | 6.2100 | 2.1661 | 0.8371 |
| 2 | SimAnneal | 1.4853 | 1.5046 | 10.4338 | 5.6700 | 2.1663 | 0.8872 |
| 3 | PSO | 1.4129 | 1.5557 | 10.5025 | 6.4200 | 2.2096 | 0.8268 |
| 4 | RandomSearch | 1.5717 | 1.8525 | 11.4466 | 7.3800 | 1.9750 | 0.7975 |
| 5 | GridSearch | 1.0000 | 1.3667 | 11.5182 | 8.8300 | 2.9399 | 0.6740 |

从表 7.2 可以看出，五种优化器在当前边界和目标权重下仍然得到相对接近的可行参数区域，但排序已不再由随机搜索主导，而是由 Nelder-Mead 以 `10.3924` 的综合得分获得最优结果。SimAnneal 与 PSO 分列第二、第三位，三者得分差均小于 `0.12`，说明目标函数景观虽然存在明显优选区域，但仍保持一定平滑性，不属于极端尖锐、只有孤立点可用的优化问题。

进一步观察各优化器对应的指标可以发现，当前最优并不等于“每一项都最好”。例如，RandomSearch 的末端误差 `1.9750 deg` 略优于 Nelder-Mead 的 `2.1661 deg`，GridSearch 的控制能耗 `0.6740 N·m·s` 也是五者中最低；但 Nelder-Mead 在调节时间、控制能耗和综合评分之间给出了更均衡的权衡，因此最终排名最高。这说明当前 PD 调参问题更适合用综合目标来选择代表参数，而不能只依赖单一指标。

这一结果同时验证了轻量化指标评估路径和本轮工程优化的价值。通过只在真正需要绘图时补跑完整时序仿真，并将快速评估阶段保持在统一动力学与扰动口径下，平台既保留了学术分析所需的可比性，又显著降低了多优化器横向比较的时间成本，使调参实验从“偶尔可跑”变为“可以反复重跑并用于研究比较”的常规能力。

### 7.3 PD 与 LADRC 控制性能比较

在完成参数整定后，平台使用统一真实惯量和统一仿真设置分别运行 PD 与 LADRC 正式仿真。表 7.3 给出了两类控制器在当前代表性场景下的主要性能指标。

| 指标 | PD | LADRC | 单位 | LADRC 相对 PD 变化 |
| --- | --- | --- | --- | --- |
| 调节时间 | 6.2800 | 1.9600 | s | +68.79% |
| 超调量 | 0.0863 | 0.0242 | deg | +71.98% |
| 稳态误差 | 1.9474 | 0.0871 | deg | +95.53% |
| 尾段 RMS 误差 | 1.9358 | 0.1490 | deg | +92.30% |
| IAE | 46.9480 | 21.6418 | deg·s | +53.90% |
| ITAE | 110.3755 | 29.5314 | deg·s² | +73.24% |
| ISU | 0.0243 | 0.1737 | N²·m²·s | -614.77% |
| 控制能耗 | 0.2857 | 0.6892 | N·m·s | -141.27% |
| 峰值单轴力矩 | 0.1909 | 0.5000 | N·m | -161.96% |
| 饱和比例 | 0.0000 | 2.0000 | % | N/A |

由表 7.3 可见，在当前统一场景下，整定后的 LADRC 在动态速度和稳态精度两方面均显著优于 PD。具体来说，LADRC 的调节时间为 `1.9600 s`，相较 PD 的 `6.2800 s` 缩短了约 `68.79%`；其末端姿态误差仅为 `0.0871 deg`，而 PD 为 `1.9474 deg`；尾段 RMS 误差也由 `1.9358 deg` 降低到 `0.1490 deg`。同时，LADRC 在 `IAE` 与 `ITAE` 上分别改善了 `53.90%` 和 `73.24%`，说明其不仅尾段误差更小，而且整个时域内的误差累积也明显更低。

然而，LADRC 的优势并非没有代价。其控制能耗由 `0.2857 N·m·s` 上升到 `0.6892 N·m·s`，输入平方积分指标 `ISU` 也从 `0.0243` 增加到 `0.1737`，峰值单轴力矩直接触及执行器上限 `0.5000 N·m`，并出现了 `2.0%` 的饱和比例，而 PD 全程未发生饱和。结合控制器结构可以解释为：LADRC 为获得更快误差衰减和更小稳态误差，需要更积极地利用控制输入，并通过 ESO 对总扰动进行实时补偿，因此其控制量级和饱和风险会相应提高。

这一现象说明，姿态控制问题中的性能比较不能只看单一指标。对于当前代表性实验，LADRC 的确在误差收敛和抗扰表现上占优，但 PD 仍体现出“控制行为保守、能耗更低、执行器边界更安全”的特点。因此，更合理的结论应是：在当前场景和当前整定结果下，LADRC 更适合作为强调快速收敛和高稳态精度的控制策略，而 PD 则更适合作为结构简单、控制代价较低的基线方案。

### 7.4 在线惯量辨识结果

前置辨识阶段分别运行了基于 ESO + RLS 的对角惯量辨识和基于增广状态 MEKF 的惯量联合估计。两类辨识结果摘要如表 7.4 所示。

| 方案 | 参考惯量 | 最终估计 | 最终误差范数 | 更新占比 |
| --- | --- | --- | --- | --- |
| RLS | `[1.6000, 1.4000, 1.0000]` | `[1.5468, 1.3011, 1.0180]` | `1.1373e-01` | `15.6%` |
| 增广状态 MEKF | `[1.6000, 1.4000, 1.0000]` | `[1.5672, 1.3866, 0.9541]` | `5.8002e-02` | `56.4%` |

从表 7.4 可以看到，两类方法都能够将惯量估计从初始值逐步修正到接近真实参考值的区域，说明当前平台已经具备对角惯量在线修正能力。但两类方案的更新特征明显不同。RLS 的更新占比为 `15.6%`，明显低于增广状态 MEKF 的 `56.4%`，这表明 RLS 在当前配置下更依赖激励门限和回归矩阵条件，只有在激励足够且创新合理时才进行有效更新；而增广状态 MEKF 的更新更为频繁，更像是在统一滤波框架内持续修正参数状态。

在最终误差范数上，增广状态 MEKF 的结果为 `5.8002e-02`，优于 RLS 的 `1.1373e-01`。这一结果说明，在当前 `fast` 场景和当前参数配置下，滤波型联合估计在最终对角惯量贴近真实值方面表现更好。但平台随后并未直接选择单一 MEKF 结果作为后续控制器模型，而是将 RLS、MEKF 与回退模型共同纳入短时闭环验证，最终选择融合名义惯量 `FUSED: [1.5569, 1.3437, 0.9861] kg·m²` 作为后续 ADRC 配置依据，并得到验证得分 `34.0834`、验证调节时间 `2.267 s` 和末端误差 `0.0062 deg`。这表明，在当前实验设置下，模型选择并不仅由辨识误差范数单独决定，还要同时考虑闭环控制中的综合鲁棒性表现。

需要指出的是，这一结果并不意味着增广状态 MEKF 在所有场景下都优于 RLS，或者融合模型在所有工况下都应被优先采用。更准确的理解是：RLS 提供了更保守、更新更稀疏的显式回归视角，MEKF 提供了更积极的参数联合修正视角，而融合模型则是在当前验证口径下兼顾两者优点的一种工程选择。这样的设计也体现了平台“辨识结果服务控制器配置”的研究思路。

### 7.5 结果讨论

综合当前实验可以得到三点认识。首先，平台已经能够以统一口径完成“前置辨识 -> 内部模型选择 -> 控制器整定 -> 正式比较 -> 结果汇总”的完整研究流程，说明“动力学—测量—估计—控制—辨识—输出”闭环链路已经打通。其次，在当前场景下，LADRC 展现出明显优于 PD 的动态与稳态误差性能，但其代价是更高的控制能耗、更大的输入平方积分以及一定程度的饱和风险，这说明姿态控制中始终存在性能与代价的权衡。再次，在线惯量辨识部分表明，平台不仅能够输出辨识轨迹，而且已经能够利用辨识结果反向服务于控制器内部模型配置，这一点是综合仿真平台相对于单模块示例程序的重要区别。

总体而言，当前结果验证了平台的研究可用性：它既能够比较控制器，也能够比较辨识方法，还能够将参数辨识结果反馈到控制器配置中形成闭环研究逻辑。对于后续正式论文完善而言，下一步最值得投入的工作不是重新搭建实验框架，而是在现有平台基础上开展更长时间、更大规模和更多随机种子的统计实验。

## 8. 讨论

### 8.1 平台能力与研究价值

从当前工作完成度来看，本文平台的主要价值不在于单独提出某种全新算法，而在于提供了一套具有研究连续性的实验基础设施。该平台将姿态动力学传播、传感器测量建模、姿态估计、控制器设计、自动调参、在线辨识和结果输出整合到统一框架内，使得不同模块不再是彼此割裂的示例程序，而是能够围绕同一对象和同一指标体系开展系统比较的研究组件。

这种平台化价值对于课程设计、毕业论文和后续科研探索都十分重要。一方面，它允许研究者在相同条件下替换控制器、替换滤波器或替换辨识器，避免因实验条件不一致而导致结论失真；另一方面，它又通过图像、表格和 Markdown 报告将实验结果沉淀下来，为论文撰写和阶段汇报节约大量重复整理成本。

### 8.2 当前实现局限

尽管平台已经具备较完整的闭环能力，但仍存在若干必须在论文中明确说明的局限。首先，当前控制闭环默认使用真实姿态和真实角速度计算控制力矩，这意味着控制器比较结果更多反映控制器在理想状态获取条件下的性能，而不是完整导航控制链路中的最终表现。虽然当前代码已经支持通过 `control_feedback_source='estimate'` 切换到滤波器输出闭环，但本文统一实验尚未围绕该模式做系统统计，因此相关结论仍需在后续工作中单独建立证据。

其次，当前在线惯量辨识仅支持对角惯量参数估计，尚未覆盖更一般的非对角惯量耦合问题。这一限制在平台验证阶段是合理的，因为对角惯量模型更利于辨识稳定性和结果解释，但若进一步追求理论完整性或复杂对象适用性，则仍需扩展参数结构。

再次，当前实验更多属于代表性场景分析，而非大规模统计研究。虽然平台已经具备开展多随机种子、多初值和多扰动批量实验的基础，但当前输出仍主要用于证明链路打通和展示代表性现象。因此，在正式论文中应避免使用“普遍优于”“显著优于”这类需要统计充分性的表述。

最后，从软件结构角度看，`main.py` 目前承担了较多实验调度、参数整定、结果汇总和报告生成职责。随着后续研究继续扩展，平台仍有必要在仿真执行、实验组织、指标计算和结果输出之间进一步拆分模块。

### 8.3 写作口径与结论边界

在正式论文写作中，最重要的不是把每一处表述都写得“更强”，而是保证表述边界与代码能力、实验证据保持一致。对于本文工作，合适的表述包括：“平台已经实现”“平台已经支持”“当前代表性实验表明”“结果体现出某种性能侧重”以及“为后续研究提供基础”。不合适的表述则包括：“提出了全新的控制理论”“在所有场景下显著优于基线方法”“已满足工程应用要求”等超出当前证据范围的结论。

论文中的讨论部分应承担一种“诚实收束”的作用，即在说明平台价值的同时，也如实交代当前口径为何不能直接外推到更复杂任务。这种写法不仅更符合学术规范，也更有助于答辩和评审阶段建立可信度。

## 9. 结论

本文围绕卫星姿态控制中的建模、测量、估计、控制与参数不确定性问题，设计并实现了一套综合仿真平台。该平台采用四元数描述姿态，利用欧拉方程建立三轴刚体姿态动力学模型，构建了包含星敏感器与陀螺仪的测量链路，实现了姿态与偏置估计、PD 与 LADRC 控制器、PD 自动调参、ADRC 带宽整定以及两类对角惯量在线辨识方法，并进一步打通了图像、CSV 数据表和 Markdown 报告的统一输出流程。

在当前统一的 `fast` 档代表性实验中，平台首先通过前置辨识得到 RLS 与增广状态 MEKF 两组惯量估计结果，并基于短时闭环验证选择融合名义惯量 `[1.5569, 1.3437, 0.9861] kg·m²` 作为后续 ADRC 配置依据；随后通过多优化器自动调参得到 PD 控制器代表参数 `Kp=1.4466`、`Kd=1.5739`，并通过带宽搜索得到 ADRC 代表参数 `omega_c=2.8000`、`omega_o=3.3950`。最终控制器比较结果表明，在当前场景下，LADRC 的调节时间由 PD 的 `6.2800 s` 降低到 `1.9600 s`，末端姿态误差由 `1.9474 deg` 降低到 `0.0871 deg`，同时在 `IAE`、`ITAE` 和尾段 RMS 误差上也有明显改善；但 PD 仍然表现出更低的控制能耗、更低的输入平方积分以及无饱和的保守控制特征。在线惯量辨识结果显示，两类方法均可实现对角惯量修正，而当前场景下增广状态 MEKF 的最终惯量误差范数小于 RLS。与此同时，本文完成的一轮代码优化还显著提高了平台研究效率，使 `ADRC_tracker_RLS` 和 `PD_optimizer_quick` 两项代表性基准分别下降到 `0.0252 s` 和 `0.2675 s`。

综上，本文工作的主要结论可以概括为三点：第一，当前代码项目已经具备支撑姿态控制研究的完整闭环实验能力；第二，PD 与 LADRC 在当前统一场景下体现出“保守低耗”与“快速高精度抗扰”两类不同性能侧重；第三，在线惯量辨识不仅能够作为独立研究对象存在，而且已经能够反向服务于控制器内部模型配置，从而形成更具研究意义的“辨识—控制联动”链路。上述结果表明，本文所构建的平台已能够作为后续正式论文深化、多场景统计实验和算法扩展研究的稳定基础。

## 10. 后续工作

### 10.1 估计姿态闭环研究

当前平台默认使用真实姿态进入控制闭环，这一设置有利于先分析控制器本体性能，但尚不能反映估计误差对闭环控制的直接影响。虽然当前代码已经支持通过 `control_feedback_source='estimate'` 将控制器输入切换为滤波器估计姿态和估计角速度，但这一模式尚未形成系统化实验章节。后续可围绕该接口开展专门研究，从而分析 MEKF 误差、星敏观测退化和陀螺偏置漂移对控制性能的耦合影响，使平台更接近完整的姿态确定与控制闭环问题。

### 10.2 更一般的惯量参数建模

当前在线辨识只面向对角惯量参数，后续可尝试将参数结构扩展到更一般的惯量矩阵形式，研究非对角耦合项在辨识稳定性和控制器内部模型更新中的影响。与此同时，还可以探索更丰富的激励设计方法，提高参数可辨识性。

### 10.3 ADRC 系统化调参与参数灵敏度研究

目前平台已具备基于已辨识惯量的 ADRC 带宽整定能力，但整定仍主要停留在工程化轻量搜索层面。后续可进一步开展 `omega_c`、`omega_o`、`td_omega`、`b0` 之间的灵敏度分析，形成更完整的 ADRC 参数研究框架，并与 PD 调参结果一起纳入统一实验口径。

### 10.4 多场景统计实验

为了使论文结论更加稳健，后续应在不同扰动水平、不同噪声条件、不同真实惯量、不同初始激励和多随机种子条件下开展批量统计实验。通过均值、方差和置信区间等统计量，可以更客观地评价不同控制器、不同辨识器在代表性场景下的总体表现，而不仅是单次运行现象。

### 10.5 平台结构重构与实验基准固化

随着研究任务增加，平台需要进一步拆分为仿真执行层、指标计算层、实验组织层和报告输出层，以降低 `main.py` 的复杂度。同时，建议建立一组固定场景基准，包括标准初始姿态、标准扰动、标准噪声和标准评价指标，使后续不同版本代码的实验结果可以在相同口径下稳定复现。
