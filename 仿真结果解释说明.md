# 仿真结果解释说明

## 1. 这份文档是干什么的

这份文档不再固定写死某一次实验的数字结果，而是说明：

- 输出目录里每类图和表代表什么
- 该怎么看 PD 与 ADRC 的差异
- 在线惯量辨识结果该怎么判断
- 优化器结果该怎么解释

这样做的好处是：

- 每次重新运行实验后，这份说明仍然适用
- 不会因为代码参数变化导致文档中的数字过期

## 2. 输出目录怎么读

运行 `python main.py` 后，通常会在 `output/` 下看到下面几类结果：

### 2.1 `output/optimization`

这里是 PD 自动调参相关结果，常见文件包括：

- `optimizer_ranking.csv`
- `optimizer_evaluated_points.csv`
- `pd_optimizer_landscape.png`
- `pd_optimizer_report_dashboard.png`

重点用途：

- 看不同优化器谁更稳
- 看最优 `Kp, Kd` 落在哪个区域
- 看目标函数是否存在明显低谷区

### 2.2 `output/simulations/pd`

这里是单次 PD 仿真结果，常见包括：

- 仿真总览图
- 控制响应图
- 摘要指标表
- 时序数据 CSV

### 2.3 `output/simulations/adrc`

这里是单次 ADRC 仿真结果，除了和 PD 类似的图之外，通常还会多出：

- 扰动估计跟踪图

### 2.4 `output/comparison`

这里是 PD 与 ADRC 的直接对比结果，最值得优先看。

### 2.5 `output/identification`

这里是在线惯量辨识相关结果，通常按 `rls` 和 `mekf` 分目录保存。

## 3. 先看哪些指标最有价值

单次仿真最常用的指标包括：

- `settle_time`
  - 调节时间
  - 越小通常表示收敛更快
- `overshoot`
  - 超调量
  - 越小说明过渡过程更平稳
- `final_error`
  - 最终姿态误差
  - 越小说明稳态精度更高
- `effort`
  - 控制能量或力矩积分
  - 越小通常表示控制代价更低
- `sat_ratio`
  - 执行器饱和比例
  - 越小越安全

不要只盯一个指标看。更合理的做法是一起看：

- 收敛快不快
- 稳态误差小不小
- 代价高不高
- 是否频繁打满执行器

## 4. PD 和 ADRC 怎么比较

### 4.1 常见现象

在这个项目里，PD 和 ADRC 往往表现出不同侧重：

- PD
  - 结构简单
  - 在模型较匹配时通常响应更直接
  - 对持续扰动的稳态鲁棒性通常不如 ADRC
- ADRC
  - 通过 ESO 估计总扰动
  - 往往在稳态误差和抗扰表现上更稳
  - 参数不合适时，调节时间可能偏长

### 4.2 看图建议

如果只有几分钟时间，建议按这个顺序看图：

1. 对比总览图
2. 姿态误差曲线
3. 控制力矩曲线
4. 扰动估计曲线

你重点要观察的是：

- 谁先进入小误差区
- 谁的尾段更稳
- 谁更容易接近力矩饱和
- ADRC 的扰动估计是否能跟上真实扰动变化

### 4.3 如何下结论

比较合理的结论模板是：

- 若 PD 的 `settle_time` 更小，但 `final_error` 和 `sat_ratio` 更差
  - 可以写成“PD 响应更快，但 ADRC 的稳态与抗扰性能更优”
- 若 ADRC 的 `final_error` 和 `effort` 都更好
  - 可以写成“ADRC 在保证控制代价可接受的情况下提升了稳态性能”
- 若两者都差不多
  - 优先选结构更简单的一方，或者继续调参提高区分度

## 5. 优化器结果怎么解释

PD 自动调参部分不是看“哪一次最幸运”，而是看：

- 平均目标值
- 不同运行之间的波动
- 最优值附近是否稳定

建议重点看这些输出：

- `optimizer_ranking.csv`
  - 看各方法平均得分、方差和代表参数
- `pd_optimizer_landscape.png`
  - 看目标函数地形是否平滑、是否有明显优区
- `pd_optimizer_report_dashboard.png`
  - 看综合收敛和参数分布

通常可以这样解释：

- `NelderMead` 更强：说明局部搜索在当前目标面上有效
- `PSO` 更强：说明群体搜索更适合当前非凸区域
- `GridSearch` 不占优：往往是预算固定时分辨率不够
- `SimAnneal` 偏弱：通常意味着给它的迭代预算还不够

## 6. 在线惯量辨识结果怎么解释

### 6.1 先看什么

优先看三件事：

- 惯量估计值是否逐步接近参考值
- 误差范数是否下降
- 更新掩码是否显示“在有激励时才更新”

### 6.2 什么时候算“辨识正常”

一般可以认为结果较正常的表现是：

- `Jxx/Jyy/Jzz` 没有明显发散
- 误差曲线总体下降
- 更新不是全程乱跳，而是和激励阶段相匹配

### 6.3 常见异常及含义

- 估计值剧烈抖动
  - 常见原因是角加速度估计噪声过大，或更新步长过激
- 长时间几乎不更新
  - 常见原因是激励不足或冻结阈值过高
- 某一轴偏差长期很大
  - 可能说明该轴激励不够，或者该轴模型误差更明显

## 7. 如何把结果写进报告

如果要写课程报告、论文或实验记录，建议按下面结构组织：

### 7.1 实验设置

写清楚：

- 仿真时长 `T`
- 时间步长 `dt`
- 是否启用星敏
- 扰动设置
- 控制器参数
- 是否启用在线惯量辨识

### 7.2 结果描述

按“速度、精度、代价、鲁棒性”四个角度写：

- 速度：看 `settle_time`
- 精度：看 `final_error`
- 代价：看 `effort` 和 `sat_ratio`
- 鲁棒性：看扰动估计和持续扰动下的尾段表现

### 7.3 结论表达

避免只写“谁更好”，更建议写成：

- “在当前扰动与初始条件下，PD 收敛更快，但 ADRC 的稳态误差更小。”
- “增广 MEKF 可以在线修正对角惯量，但对激励条件和滤波参数较敏感。”
- “轻量指标模式显著缩短了 PD 调参时间，使多优化器横评更可行。”

## 8. 实用建议

- 想看最终结论时，优先跑 `profile='full'`
- 想验证代码和流程时，先跑 `profile='fast'`
- 想做最终成文图表，建议保留同一组随机种子和参数设置，避免实验口径不一致
