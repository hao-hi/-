# 仿真结果解释说明（当前代码版本）

## 1. 本次结果来源
- 脚本：`main.py` 中的 `compare_pd_gain_optimizers()` + `simulate_attitude_control()`
- 评估设置（用于快速验证）：
  - 调参仿真：`T=8.0s, dt=0.03, umax=0.5`
  - 优化器对比：`benchmark_runs=2, objective_eval_seeds=[0,1], quick=True`
  - 控制器对比：`T=10.0s, dt=0.03, use_star_tracker=True`

说明：这是“快速基准”结果，主要用于方法有效性验证；最终报告建议用 `quick=False` 再跑一次全量结果。

## 2. 五优化器调参结果（实测）

### 2.1 排名（按 `Score(mean)`）
1. `NelderMead`：`Kp=5.0869, Kd=0.5832, Score(mean)=13.196`
2. `PSO`：`Kp=4.1290, Kd=0.5560, Score(mean)=13.222`
3. `RandomSearch`：`Kp=3.5660, Kd=0.3700, Score(mean)=13.270`
4. `GridSearch`：`Kp=8.0000, Kd=1.0500, Score(mean)=13.417`
5. `SimAnneal`：`Kp=7.6250, Kd=1.7430, Score(mean)=13.902`

### 2.2 结果解读
- `NelderMead` 在当前参数区间内表现最稳定，均值最优且方差极小。
- `PSO` 的最优值接近 `NelderMead`，但 run 间波动略高。
- `SimAnneal` 在快速模式下未充分收敛，均值明显偏高。
- `GridSearch` 依赖网格分辨率；在快速模式下容易出现“离散最优”限制。

## 3. PD 与 ADRC 对比结果（以最优 PD 参数）

### 3.1 实测指标
- 最优 PD 参数：`Kp=5.0869, Kd=0.5832`
- PD：
  - `settle_time = 0.390 s`
  - `overshoot = 14.234 deg`
  - `final_error = 0.5919 deg`
  - `effort = 0.344`
  - `sat_ratio = 0.0030`
- ADRC（`b0=[20,20,20], omega_c=2.5, omega_o=8.0`）：
  - `settle_time = 3.210 s`
  - `overshoot = 16.700 deg`
  - `final_error = 0.3618 deg`
  - `effort = 0.326`
  - `sat_ratio = 0.0000`

### 3.2 结论解释
- 当前设置下：
  - **PD 更快**（调节时间显著更短）。
  - **ADRC 静差更小、饱和更少**（抗扰稳态表现更好）。
- 这符合两者典型特性：
  - PD 在匹配工况下响应快；
  - ADRC 在扰动存在时稳态鲁棒性更强。
